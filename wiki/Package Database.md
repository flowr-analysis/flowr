# Package Database

flowR ships a database of CRAN package exports so it can resolve calls into the packages you load.
After `library(ggplot2)`, a call to `ggplot()` resolves to `ggplot2::ggplot`. This also backs
qualified names (a bare `map()` after `library(purrr)` is `purrr::map`, not the `maps` plot), the
[dependencies and call-context queries](https://github.com/flowr-analysis/flowr/wiki/query-api), and the [`undefined-symbol` rule](https://github.com/flowr-analysis/flowr/wiki/linter).

## Configuration

The exports come from [`versions:pkgdb`](https://github.com/flowr-analysis/flowr/tree/main/src/project/plugins/plugin-registry.ts), which reads the bundled database (on by default;
see [configuring flowR](https://github.com/flowr-analysis/flowr/wiki/interface#configuring-flowr)).

*Which* version's exports are resolved is decided by the version-reading plugins that pin the packages
used in a project. These are only examples - more may be registered:

- [`versions:description`](https://github.com/flowr-analysis/flowr/tree/main/src/project/plugins/plugin-registry.ts) reads a package's `DESCRIPTION` (`Depends` / `Imports`, [src/project/plugins/package-version-plugins/flowr-analyzer-package-versions-description-file-plugin.ts](https://github.com/flowr-analysis/flowr/tree/main//src/project/plugins/package-version-plugins/flowr-analyzer-package-versions-description-file-plugin.ts)).
- [`versions:renv`](https://github.com/flowr-analysis/flowr/tree/main/src/project/plugins/plugin-registry.ts) and [`versions:rv`](https://github.com/flowr-analysis/flowr/tree/main/src/project/plugins/plugin-registry.ts) read a project's `renv.lock` / `rv.lock`
  to pin exact versions ([src/project/plugins/package-version-plugins/flowr-analyzer-package-versions-lockfile-plugin.ts](https://github.com/flowr-analysis/flowr/tree/main//src/project/plugins/package-version-plugins/flowr-analyzer-package-versions-lockfile-plugin.ts)).

So `library(pkg)` resolves the pinned version's exports. To override or extend the bundle, hand
[`versions:pkgdb`](https://github.com/flowr-analysis/flowr/tree/main/src/project/plugins/plugin-registry.ts) extra sources (file paths, parsed objects, or `http(s)` URLs).


```ts
function usePackageDatabase(parser: KnownParser) {
	const pkgdb = new FlowrAnalyzerPackageVersionsPkgDbPlugin('/path/to/pkgdb-all.json.br');
	return new FlowrAnalyzerBuilder().setParser(parser).registerPlugins(pkgdb).build();
}
```

## Format

One JSON object (`flowr-pkgdb`, schema 4) with a pooled string table.
Reader and writer both live in [src/project/plugins/package-version-plugins/pkgdb.ts](https://github.com/flowr-analysis/flowr/tree/main//src/project/plugins/package-version-plugins/pkgdb.ts).
