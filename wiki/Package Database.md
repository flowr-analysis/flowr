# Package Database

flowR ships a database of CRAN package exports so it can resolve calls into the packages you load.
After `library(ggplot2)`, a call to `ggplot()` resolves to `ggplot2::ggplot`. This also backs
qualified names (a bare `map()` after `library(purrr)` is `purrr::map`, not the `maps` plot), the
[dependencies and call-context queries](https://github.com/flowr-analysis/flowr/wiki/query-api), and the [`undefined-symbol` rule](https://github.com/flowr-analysis/flowr/wiki/linter).

## Configuration

Three plugins provide this, all on by default (see [configuring flowR](https://github.com/flowr-analysis/flowr/wiki/interface#configuring-flowr)):

- [`versions:pkgdb`](https://github.com/flowr-analysis/flowr/tree/main/src/project/plugins/plugin-registry.ts) reads the bundled database.
- [`versions:renv`](https://github.com/flowr-analysis/flowr/tree/main/src/project/plugins/plugin-registry.ts) and [`versions:rv`](https://github.com/flowr-analysis/flowr/tree/main/src/project/plugins/plugin-registry.ts) read a project's `renv.lock` / `rv.lock`
  to pin package versions, so `library(pkg)` resolves the pinned version's exports ([src/project/plugins/package-version-plugins/flowr-analyzer-package-versions-lockfile-plugin.ts](https://github.com/flowr-analysis/flowr/tree/main//src/project/plugins/package-version-plugins/flowr-analyzer-package-versions-lockfile-plugin.ts)).

To override or extend the bundle, hand [`versions:pkgdb`](https://github.com/flowr-analysis/flowr/tree/main/src/project/plugins/plugin-registry.ts) extra sources (file paths, parsed
objects, or `http(s)` URLs).


```ts
function usePackageDatabase(parser: KnownParser) {
	const pkgdb = new FlowrAnalyzerPackageVersionsPkgDbPlugin('/path/to/pkgdb-all.json.br');
	return new FlowrAnalyzerBuilder().setParser(parser).registerPlugins(pkgdb).build();
}
```

## Tiers

Three tiers, `tiny`, `latest`, and `all` (every package, version, and identifier), are built from
CRAN's history. npm and Docker ship the smaller tiers; download `all` for full coverage and
version-aware resolution. `defaultPkgDbPath()` picks the richest tier available and falls back to `latest`.

## Format

One JSON object (`flowr-pkgdb`, schema 4) with a pooled string table. The `all` tier also
pools export lists and can carry optional definition locations. CRAN source URLs are reconstructed on
load. Reader and writer both live in [src/project/plugins/package-version-plugins/pkgdb.ts](https://github.com/flowr-analysis/flowr/tree/main//src/project/plugins/package-version-plugins/pkgdb.ts).
