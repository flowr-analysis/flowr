This rule is a `best-effort` rule.
 
Detects uses of eval-like functions whose inputs are not statically constant. Prints the computed input-sources for the eval and flags usages that depend on non-constant/trusted inputs.\
_This linting rule is implemented in <a href="https://github.com/flowr-analysis/flowr/tree/main//src/linter/rules/problematic-eval.ts#L44">src/linter/rules/problematic-eval.ts</a>._


### Configuration

Linting rules can be configured by passing a configuration object to the linter query as shown in the example below.
The `problematic-eval` rule accepts the following configuration options:

- <a href="https://github.com/flowr-analysis/flowr/tree/main//src/linter/rules/problematic-eval.ts#L39"><code><span title="All calls that should be considered to be valid eval entry points, this will be interpreted as a Regex!">considerAsEval</span></code></a>\
All calls that should be considered to be valid eval entry points, this will be interpreted as a Regex!

### Examples


```r

function(x) {
	eval(x)
}
```


The linting query can be used to run this rule on the above example:




```json
[ { "type": "linter",   "rules": [ { "name": "problematic-eval",     "config": {} } ] } ]
```
